# Design: Events & Calendar

## Technical Approach

Layered implementation following ManteMap's existing patterns: Prisma model → Zod validation → Repository → Service → API Route → TanStack Query hooks → Client Components. FullCalendar loaded via `next/dynamic` to keep the ~200KB bundle out of the main chunk. RRULE stored as a string field, expanded server-side via `rrule.js` for date-range queries.

## Architecture Decisions

| Decision | Choice | Alternatives | Rationale |
|----------|--------|--------------|-----------|
| RRULE storage | `String` field on Event model | Separate `RecurrenceRule` model, JSON field | Simplest. RRULE is a single RFC 5545 string. `rrule.js` parses it directly. No relational benefit from normalization. |
| Calendar library | FullCalendar (dynamic import) | react-big-calendar, custom canvas | FullCalendar has the richest view system (day/week/month), built-in RRULE plugin, and event source API. Dynamic import mitigates bundle cost. |
| Event source merging | Single API endpoint returns manual + expiration events | Separate endpoints, client-side merge | Single endpoint simplifies FullCalendar's `events` callback. Server has full context for both sources. |
| Expiration events | Virtual (computed from `Document.expiresAt`) | Persisted Event rows with `isAutoGenerated` flag | Avoids data duplication and sync issues. Expirations are read-only projections. |
| Date handling | UTC storage, `date-fns-tz` for display | Store local timezone offset in model | UTC is the standard. Timezone conversion is a presentation concern. |

## Data Flow

```
Browser (FullCalendar)
  │ fetchEvents(start, end)
  ▼
GET /api/projects/[projectId]/events?start=...&end=...
  │
  ▼
event-service.getEventsInRange(projectId, start, end, userId)
  ├─→ event-repository.findEventsByDateRange()     ← manual events
  └─→ document-repository.findExpiringDocuments()   ← virtual events
  │
  ▼  merge + map to FullCalendar EventInput format
  │
  ▼
FullCalendar renders events in day/week/month views
```

## Interfaces / Contracts

```prisma
model Event {
  id          String    @id @default(cuid())
  projectId   String
  itemId      String?
  title       String
  description String?
  startAt     DateTime  @db.Timestamptz
  endAt       DateTime  @db.Timestamptz
  allDay      Boolean   @default(false)
  rrule       String?           // RFC 5545 RRULE string
  color       String?           // hex color for calendar display
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  project Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  item    Item?   @relation(fields: [itemId], references: [id], onDelete: SetNull)

  @@index([projectId, startAt])
  @@index([projectId, endAt])
  @@index([itemId])
  @@map("events")
}
```

```typescript
// packages/validation/src/event.ts
export const createEventSchema = z.object({
  title: z.string().trim().min(1).max(200),
  description: z.string().max(2000).optional(),
  startAt: z.string().datetime(),
  endAt: z.string().datetime(),
  allDay: z.boolean().optional().default(false),
  itemId: z.string().cuid().optional(),
  rrule: z.string().optional(),  // validated by rrule.js on server
  color: z.string().regex(/^#[0-9a-fA-F]{6}$/).optional(),
});

export const eventFilterSchema = z.object({
  start: z.string().datetime(),
  end: z.string().datetime(),
});
```

```typescript
// FullCalendar event mapping
interface CalendarEventSource {
  id: string;
  title: string;
  start: string;      // ISO 8601
  end: string;        // ISO 8601
  allDay: boolean;
  color?: string;
  extendedProps: {
    type: 'manual' | 'document-expiration';
    eventId?: string;
    documentId?: string;
    itemId?: string;
  };
}
```

## Testing Strategy

| Layer | What to Test | Approach |
|-------|-------------|----------|
| Unit | Zod schemas (createEventSchema, eventFilterSchema) | Vitest, valid/invalid inputs |
| Unit | Event service (CRUD, date-range merge, RRULE expansion) | Vitest with Prisma mock |
| Unit | Event repository (queries, date filtering) | Vitest with Prisma mock |
| Component | CalendarView renders, navigates views, loads events | Vitest + React Testing Library |
| Component | EventFormDialog validates, submits, handles RRULE picker | Vitest + React Testing Library |
| Integration | API routes (auth, validation, error cases) | Vitest + supertest pattern |

## Threat Matrix

N/A — no routing, shell, subprocess, VCS/PR automation, executable-file classification, or process-integration boundary.

## Migration / Rollout

1. Add Event model via `prisma db push` (dev) or new migration (production, after ADR-005 baseline)
2. No feature flag needed — calendar page is a new route, no existing behavior modified
3. Document expiration events are virtual — no data migration required
